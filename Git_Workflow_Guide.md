# สรุปขั้นตอนและคำสั่ง Git สำหรับการนำโปรเจกต์ขึ้น GitHub

ไฟล์นี้รวบรวมขั้นตอนและคำสั่ง Git ทั้งหมดที่เราใช้ในการนำโปรเจกต์ที่มีอยู่แล้วในเครื่อง (Local) ขึ้นไปยัง GitHub (Remote) เรียงตามลำดับครับ

## ลำดับขั้นตอนการนำโค้ดขึ้น GitHub

1. **สร้าง Git Repository ในเครื่อง** เพื่อเริ่มระบบติดตามไฟล์
2. **เชื่อมต่อกับ GitHub** ด้วย Link ของ Repository ที่เตรียมไว้
3. **ตั้งค่าผู้ใช้งาน (เฉพาะครั้งแรก)** ให้ Git รู้ว่าใครเป็นคนบันทึกโค้ด
4. **บันทึกการเปลี่ยนแปลง (Add & Commit)** เตรียมและบันทึกโค้ดลงระบบ Git ในเครื่อง
5. **อัปโหลด (Push)** นำโค้ดที่บันทึกแล้วส่งขึ้นไปเก็บบน GitHub

---

## คำอธิบายคำสั่งที่ใช้งาน (เรียงตามลำดับ)

### 1. การเริ่มต้นและตั้งค่าระบบ
* `git init`
  * **ความหมาย:** สั่งให้โฟลเดอร์ปัจจุบันกลายเป็น Git Repository ระบบจะสร้างโฟลเดอร์ซ่อนชื่อ `.git` ขึ้นมาเพื่อเริ่มติดตามการเปลี่ยนแปลงไฟล์
* `git remote add origin https://github.com/...`
  * **ความหมาย:** เป็นการผูกโปรเจกต์ในเครื่องของเราเข้ากับโปรเจกต์บนเว็บ GitHub โดยตั้งชื่อเล่นให้ปลายทางบนเว็บว่า `origin`
* `git branch -M main`
  * **ความหมาย:** เปลี่ยนชื่อสาขาหลัก (Branch) ในเครื่องจากชื่อเดิม (มักจะเป็น `master`) ให้เป็นชื่อ `main` เพื่อให้ตรงกับมาตรฐานใหม่ของ GitHub

### 2. การระบุตัวตน (สำหรับเครื่องที่ยังไม่เคยใช้งาน Git)
* `git config --global user.email "อีเมลของคุณ"`
* `git config --global user.name "ชื่อของคุณ"`
  * **ความหมาย:** ตั้งค่าอีเมลและชื่อผู้ใช้ให้กับระบบ Git ในเครื่อง เพื่อให้เวลาบันทึกประวัติ (Commit) ระบบจะได้บันทึกได้ว่า "ใคร" เป็นคนแก้ไขโค้ดชุดนั้นๆ

### 3. การบันทึกและอัปโหลดโค้ด
* `git add .`
  * **ความหมาย:** นำไฟล์ทั้งหมดในโฟลเดอร์ (`.`) ที่มีการสร้างใหม่, แก้ไข หรือลบ เข้าสู่สถานะเตรียมบันทึก (Staging Area) 
* `git commit -m "Initial commit"`
  * **ความหมาย:** สั่งบันทึกไฟล์ที่เตรียมไว้ทั้งหมดลงในประวัติของ Git (ในเครื่องเรา) โดย `-m` คือการแนบข้อความอธิบายว่าการบันทึกครั้งนี้ทำอะไรไปบ้าง
* `git push -u origin main -f`
  * **ความหมาย:** 
    * `push`: ดันไฟล์และประวัติทั้งหมดที่บันทึกไว้ขึ้นไปยัง GitHub (`origin`)
    * `-u`: สั่งให้ระบบจำไว้ว่ารอบถัดไปถ้าเราพิมพ์แค่ `git push` ให้ส่งไปที่ `origin` สาขา `main` อัตโนมัติ
    * `-f`: (ย่อมาจาก `--force`) คือการ **บังคับทับ** โค้ดบน GitHub ด้วยโค้ดจากในเครื่องของเรา (เราใช้คำสั่งนี้เพื่อเอาโค้ดในเครื่องไปทับไฟล์ README.md ว่างๆ บน GitHub ที่เพิ่งสร้างมาตอนแรกครับ)
