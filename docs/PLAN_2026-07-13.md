# แผนปรับปรุง & เตรียมขึ้น Production — Project Document (Document ES)

> จัดทำ 2026-07-13 · ตรวจสอบจากโค้ดจริง (ไม่ใช่แค่เอกสารเดิม) · `go build ./...` และ `tsc --noEmit` = 0 error

## สรุปภาพรวม
โค้ด **พร้อมระดับ production เป็นส่วนใหญ่แล้ว** — ไม่มี blocker ที่ทำให้ระบบใช้งานไม่ได้
สิ่งที่ทำไว้ดี (ยืนยันจากโค้ด): graceful shutdown, HTTP timeouts, pgxpool tuning, CORS env-gated,
storage factory, parameterized SQL ทั้งหมด, RBAC 2 ชั้น, JWT refresh rotation (hash+revoke),
path ไฟล์เป็น UUID (กัน traversal), SSE + token-rotation reconnect, FK ON DELETE CASCADE,
Acknowledge race กันด้วย unique index + ON CONFLICT, soft-delete user + revoke session

งานที่เหลือ = **hardening (โค้ด)** + **infra/config ตอน deploy**

## 🔴 กลุ่ม A — Hardening ก่อนขึ้น production (งานโค้ด)

| # | เรื่อง | จุดในโค้ด | แนวทาง | สถานะ |
|---|---|---|---|---|
| A1 | ไม่มี Rate Limiting | `/auth/login`, `/auth/refresh`, `/auth/azure/exchange` | `middleware/ratelimit.go` token-bucket ต่อ IP (30/นาที, burst 10) | ✅ ทดสอบ runtime: 429 หลัง burst |
| A2 | ไม่มี Security Headers | `middleware/security.go` + `next.config.mjs` headers() | X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy | ✅ ทดสอบ runtime: header ครบ |
| A3 | Upload validate แค่ client + Content-Disposition ดิบ | `document.go` `validateUpload`/`contentDisposition` | whitelist ext (pdf/jpg/jpeg/png) ฝั่ง server + sanitize header (RFC 5987) | ✅ build ผ่าน |
| A4 | ไม่มี Error Boundary | `app/error.tsx` + `app/global-error.tsx` | fallback ภาษาไทย กันจอขาว | ✅ build ผ่าน |
| A5 | Next.js 14.2.5 มี CVE สะสม | `package.json` | อัปเป็น **14.2.35** (patch สาย 14.2 — ไม่ breaking) | ✅ ติดตั้ง+build ผ่าน |
| A6 | SSE โดน WriteTimeout 5 นาที ตัด | `notifications.go` Stream | `http.NewResponseController(w).SetWriteDeadline(time.Time{})` | ✅ build ผ่าน |

> CVE Next ที่ไม่กระทบ: i18n / middleware / beforeInteractive / CSP-nonce (แอปไม่ได้ใช้ — ไม่มี `middleware.ts`, ไม่มี i18n)
> **หมายเหตุ A5:** อัป 14.2.35 เก็บ patch ทั้งหมดของสาย 14.2 แล้ว แต่ `npm audit` **ยังไม่ 0** — เหลือ advisory กลุ่ม DoS/RSC/rewrite-smuggling ที่ Vercel patch เฉพาะใน Next 15/16 (major breaking) ส่วนใหญ่ไม่กระทบการใช้งานจริง (image ใช้แค่ avatar ภายใน) — การอัป Next 16 เป็นการตัดสินใจแยก **อย่า** `audit fix --force`

## 🟠 กลุ่ม B — Infra / Config ตอน Deploy
อ้างอิงหลักที่ `docs/DEPLOY_CHECKLIST.md` (TLS, Azure redirect URI, binary+process manager, nginx SSE, backup) + เพิ่มที่พบใหม่:
- **Frontend Dockerfile ขาด `NEXT_PUBLIC_BASE_PATH` ARG** (ส่งแค่ API_BASE_URL) → build ไม่ติด basePath
- **Docker ไม่มี HEALTHCHECK** ทั้ง backend/frontend (backend มี `/healthz` ใช้ได้เลย)
- **docker-compose ตั้ง JWT secret default อ่อน** (`dev-...-change-me`) → prod ต้อง require env จริง (ตัด `:-default`)
- **avatars volume ไม่ถูก mount** ใน compose (`storage/avatars` หายเมื่อ rebuild) → เพิ่ม volume
- backend Dockerfile รัน `go mod tidy` ตอน build + รันเป็น root → pin deps + non-root user
- `.env.local.example` ไม่ครบ (ขาด `BACKEND_URL`, `NEXT_PUBLIC_BASE_PATH`)
- ตั้ง log rotation สำหรับ prod
