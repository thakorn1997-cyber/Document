## ✅ งานที่ทำวันนี้

### 1. แก้บั๊ก Login — ฟอร์ม local แว้บตอน refresh
- `login/page.tsx` — เดิม `methods` เริ่มเป็น `null` → `localEnabled` เดาเป็น `true` → ฟอร์ม local โผล่ก่อนทุกครั้งแม้ปิดอยู่
- แก้: แสดง `<MusicLoader/>` จนกว่า `/auth/methods` โหลดเสร็จ แล้วค่อย render ฟอร์มตามค่าจริง
- `.catch()` fallback (`local_enabled:true`) คงไว้กันล็อกตายเมื่อ API ล่ม

### 2. แก้บั๊กสำคัญ — ข้อมูล user ค้างข้ามการ login (Azure → local)
- **สาเหตุ:** logout ล้างแค่ `tokenStore` แต่ **ไม่ล้าง React Query cache** (`QueryClient` มีชีวิตตลอดแท็บ, `staleTime 30s`) → cache `["me"]` ของ Azure ค้าง → login local ใหม่เห็นเป็น user Azure
- แก้: `logout()` (`(protected)/layout.tsx`) + login สำเร็จทั้ง local/Azure (`login/page.tsx`) เรียก `qc.clear()`
- Backend revoke refresh token ถูกต้องอยู่แล้ว ไม่ต้องแก้

### 3. หน้า Report ใหม่ (`/reports`) — ฟีเจอร์หลักของวัน
เข้าได้ **ทุก Role** (NAV key `report`, ไม่ใช่ adminOnly). ไม่เพิ่ม backend endpoint — ใช้ `documentApi.list({size:500})` + `settingsApi.get()`
- **คอลัมน์ aging "จำนวนวัน"** — นับวันปฏิทินจากวันติดตั้ง; UAT/UAI = Passed → ล็อกค่า (`date − install`), ยังไม่ผ่าน → นับถึงวันนี้; edge case (ไม่มีวันติดตั้ง / Passed แต่ไม่มีวันที่) → `—`
- **ตัวกรองประเภท** UAI / UAT / ทั้งหมด (default = ทั้งหมด) → คอลัมน์+aging+สรุป+export อิงประเภทที่เลือก
- **UI filter = chip แบบ B+C** — ชิปกลม, ไฮไลต์ฟ้าเมื่อกำลังกรอง, วันที่เป็น preset (7/30/90/กำหนดเอง)
- **Export dropdown** — Excel (SheetJS, dynamic import) + PDF (เปิดหน้าพิมพ์ Sarabun, จัด align หัว/ข้อมูลตรงกัน) ตาม filter ปัจจุบัน
- **เกณฑ์สี aging กำหนดเองได้** — เก็บใน `app_settings.report_aging` `{warn_days,late_days}` (default 8/30) ตั้งที่ Settings › รายงาน (admin), อ่านทุก role
- ไฟล์ใหม่: `reports/page.tsx`, `settings/tabs/ReportSettingsTab.tsx` · แก้: `settings/page.tsx`, `MenuPermissionTab.tsx`, `layout.tsx`, `endpoints.ts`
